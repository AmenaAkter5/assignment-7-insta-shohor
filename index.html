JavaScript Concepts

How does JavaScript work?

JavaScript is a single threaded, interpreted, lightweight and the world most popular programming language. It can only handle one operation at a time, or one line of code at a time. It's an interpreted programming language that, like most scripting languages, makes use of dynamic typing, which verifies type safety during runtime.

JavaScript has no compilation step. It works with own JIT compiler in every web browsers. JIT is abbreviated form of Just In Time. Basically in JavaScript JIT is such a process where engine convert JavaScript code to machine code (bytecode). So that machine easily understand JavaScript Code and he can deliver more faster output to the user. Like, the Google Chrome browser uses a JIT engine called V8 for converting JavaScript code to bytecode.

LocalStorage Vs sessionStorage

We have a number of options for caching or storing information on the client browser. localStorage and sessionStorage are two ways of them.

From the name localStorage and sessionStorage we can understand that they store somethings to use them in near future. Yes, browser stores data on client's computer by localStorage and sessionStorage to get user informations and use them to operate different operations in near future.

Data storing format is same in localStorage and sessionStorage. They store the data as key/value pairs in a web browser.

Although they are same in format but there are differences between localStorage and sessionStorage in their persistency and usages.

LocalStorage SessionStorage
LocalStorage lasts until the user deletes it. SessionStorage get cleared when the page or tab is closed.
Data in localStorage has no expiration date. Data in sessionStorage is temporay.
Changes are saved and available for all current and future visits to the site. For sessionStorage, changes made are saved and available for the current
page in that tab until it is closed.
